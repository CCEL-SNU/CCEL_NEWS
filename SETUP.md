# CCEL Weekly News - Setup Guide
# =============================

## Prerequisites
- Windows PC/server on SNU network (always on)
- Python 3.9+ installed
- Git installed
- GitHub account
- Anthropic API key

## Step 1: Clone and Install

```powershell
# Clone the repository
git clone https://github.com/YOUR_USERNAME/ccel-daily-news.git
cd ccel-daily-news

# Create virtual environment (recommended)
python -m venv backend\venv
backend\venv\Scripts\activate

# Install dependencies
pip install -r backend\requirements.txt
```

## Step 2: Configure

1. Edit `backend/config.yaml`:
   - Update Semantic Scholar author IDs for each group PI
     (Find IDs at: https://www.semanticscholar.org/)
   - Adjust keywords to match CCEL's current research focus
   - Modify RSS feeds if needed

2. Set environment variables:
   ```powershell
   # Option A: Set permanently in System Environment Variables
   # Settings > System > About > Advanced system settings > Environment Variables
   # Add: ANTHROPIC_API_KEY = sk-ant-xxxxx

   # Option B: Edit run_daily.bat and set the key there
   ```

## Step 3: Test Run

```powershell
cd backend

# Test collection only (no API cost)
python run_daily.py --collect

# Test full pipeline (uses Claude API credits)
python run_daily.py

# Check output
type data\news.json
```

## Step 4: Setup Git Push

```powershell
# In the repo root
git remote set-url origin https://github.com/YOUR_USERNAME/ccel-daily-news.git

# Configure git credentials (so push works without password prompt)
git config credential.helper store
git push origin main  # Enter credentials once, they'll be saved
```

## Step 5: Schedule with Windows Task Scheduler

1. Open Task Scheduler: press Win+R, type `taskschd.msc`
2. Click "Create Basic Task..."
3. Name: "CCEL Weekly News"
4. Trigger: Daily, Start time: 07:00
5. Action: Start a program
   - Program/script: `cmd.exe`
   - Arguments: `/c "C:\path\to\ccel-daily-news\backend\run_daily.bat"`
   - Start in: `C:\path\to\ccel-daily-news\backend`
6. Check "Open Properties dialog" > Finish
7. In Properties:
   - Check "Run whether user is logged on or not"
   - Check "Run with highest privileges"
   - Settings tab: Allow task to be run on demand

## Step 6: Setup GitHub Pages

1. Go to your GitHub repo > Settings > Pages
2. Source: GitHub Actions
3. The deploy.yml workflow will auto-deploy on each push

## Step 7: Frontend Setup (if building locally)

```powershell
cd frontend
npm install
npm run build
```

The React app reads from `public/data/news.json` which is
auto-generated by the backend pipeline.

## Verification

After the first scheduled run:
1. Check `backend/logs/` for run logs
2. Check `backend/data/news.json` for output
3. Check GitHub repo for the pushed commit
4. Visit your GitHub Pages URL to see the live site

## Troubleshooting

### Papers not downloading
- Verify the server is on SNU network (check IP at whatismyip.com)
- Some publishers may need additional headers; check logs for 403 errors
- Reduce `max_papers_per_day` if getting rate limited

### Claude API errors
- Verify ANTHROPIC_API_KEY is set correctly
- Check API credit balance at console.anthropic.com
- Reduce batch size if hitting rate limits

### Git push fails
- Run `git push origin main` manually to diagnose
- Check that credentials are stored: `git config credential.helper`
- For SSH: ensure SSH key is added to GitHub

### Task Scheduler not running
- Check Task Scheduler History (right-click task > Properties > History)
- Ensure "Run whether user is logged on or not" is checked
- Try running run_daily.bat manually first
